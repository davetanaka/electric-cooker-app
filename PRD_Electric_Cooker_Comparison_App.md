# PRD: 電気調理鍋 横断比較Webアプリ

**バージョン**: 1.0
**作成日**: 2026-03-06
**作成者**: Dave Tanaka
**ステータス**: Draft
**Planning Sheet**: AI Planning Sheet by Dave v4.0 に基づく

---

## 1. 概要

電気調理鍋6社の製品を、公式スペック（Layer 1）とSNS・コミュニティの生の声（Layer 2）の両面から横断比較できるWebアプリ。カタログ廃止・アフィリエイト汚染という消費者の課題を、データとAIで解決する。

### 1.1 キーメッセージ

**「カタログに載っていない情報も含めた、消費者のための横断比較」**

### 1.2 背景

- ヨドバシカメラ・アキバ店で電気調理鍋を見に行ったところ、カタログがほぼ廃止されていた
- ネットで調べてもアフィリエイト記事ばかりで、消費者目線の横断比較ができない
- メーカーごとにスペック表示がバラバラで、自力での比較に膨大な時間がかかる
- 自分が困ったから、自分で作ることにした

---

## 2. ユーザーペルソナ

**ターゲット**: 電気調理鍋を検討しているすべての消費者

具体的には以下のような人を含む:

- **店頭で困っている人**: カタログがなく、店員に聞いても横断比較の情報が得られない
- **忙しい共働き世帯**: ネットの情報を精査する時間がなく、効率的に候補を絞りたい
- **納得して買いたい人**: 家族を説得できる客観的な判断材料がほしい
- **生活の質を上げたい人**: 時短調理が目的で、製品選びはその手段

### 2.1 ユーザーゴール

- 自分の条件に合った1台を、納得して選びたい
- 買った後に後悔しないよう、事前に比較検討を済ませたい
- 家族を説得できる客観的な判断材料がほしい
- 時短調理で生活の質を上げたい（製品選びはその手段）

### 2.2 ペインポイント（優先度順）

1. **カタログ廃止**: 店頭に比較材料がない
2. **アフィリエイト汚染**: ネットの情報が信頼できない
3. **スペック表示の不統一**: メーカーごとにバラバラで横比較できない
4. **購入後情報の不透明さ**: SNSの口コミやメンテナンス実態など「買った後」の情報が見えない

## 3. 成功の定義

### 3.1 CTA（ユーザーに期待するアクション）

自分の優先条件でフィルター絞り込み → AIチャットに質問 → **1台に決められる状態になる**

### 3.2 検証方法

YouTube視聴者テスト:
- 動画内でURLを公開し「使ってみてください」と呼びかけ
- コメント欄・コミュニティ投稿でフィードバックを収集
- Vercelのアクセスログで利用状況を確認

## 4. 機能要件

### 4.1 根拠の柱（価値を支える4要素）

| 優先度 | 機能 | 説明 |
|--------|------|------|
| 1 | スペック横断比較（Layer 1） | 6社の公式スペックデータ約30項目を統一フォーマットで表示。圧力有無、容量、Wi-Fi対応、かき混ぜ機能、内蔵レシピ数、価格帯など |
| 2 | 生の声（Layer 2） | SNS・コミュニティ・メンテナンス実態。カタログにない購入後の情報 |
| 3 | 条件フィルター | 自分の優先条件（予算、容量、圧力有無、Wi-Fi等）で絞り込み |
| 4 | AIチャット | 6社のデータを踏まえた個別質問対応。「3人家族で圧力必須、予算3万円」のような自然言語に対応 |

### 4.2 対象製品（6社）

1. シャープ ヘルシオ（ホットクック）
2. パナソニック（オートクッカー ビストロ）
3. ティファール（クックフォーミー）
4. アイリスオーヤマ（ヘルシープラス）
5. 象印（STAN.）
6. サーモス（シャトルシェフ ※保温調理器）

### 4.3 データ構造

**Layer 1（スペック）**: 約30項目
- 基本: メーカー名、製品名、型番、価格帯、実売価格
- 機能: 圧力有無、最大圧力(kPa)、かき混ぜ機能、低温調理、無水調理、蒸し調理、炊飯
- 容量: 調理容量(L)、満水容量(L)、対応人数目安
- 接続: Wi-Fi対応、アプリ連携、音声操作対応
- レシピ: 内蔵レシピ数、アプリレシピ数
- サイズ: 幅×奥行×高さ(mm)、重量(kg)
- 付属品: 内鍋素材、付属品一覧
- 消費電力、コード長

**Layer 2（SNS・コミュニティ）**:
- ユーザーコミュニティ有無・規模（ホットクックのみ約2万人規模）
- Instagram映え度
- メンテナンス実態（パーツ洗いやすさ、消耗品コスト）
- 購入後サポート体制
- よくある不満点

## 5. 非機能要件 / ガードレール

### 5.1 情報ポリシー

- 使用情報: **Public＋General**（各社公式サイト・公開スペック・公開SNS・小売店からの一般情報）
- 数値には**必ず出典をつける**
- **アフィリエイトリンクを一切入れない**
- 特定メーカーを優遇・批判しない中立的な立場を保つ

### 5.2 組織上の位置づけ

- **個人プロジェクト**。会社のリソースは使用しない。承認フロー不要

### 5.3 テックスタック

| カテゴリ | 技術 |
|----------|------|
| フレームワーク | Next.js (App Router) |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |
| UIコンポーネント | shadcn/ui |
| AI | Claude API |
| ホスティング | Vercel |

### 5.4 セキュリティ

- APIキーは `.env` に格納し、フロントエンドに露出しない
- ユーザーデータをサーバーに保存しない

### 5.5 開発ルール

- **Git管理**: 機能単位でこまめにcommit。mainブランチに直接pushしない
- **サブエージェント活用**: 調査と実装を分離。1タスク1エージェント
- **コード品質**: プレースホルダー禁止。完全なコードのみ記述

## 7. UXガイドライン

- **トーン**: 一般消費者向け。親しみやすさ＋データ
- **専門用語**: 使う場合は必ず補足説明を添える（例：「kPa（キロパスカル＝圧力の単位。数字が大きいほど高圧）」）
- **中立性**: どのメーカーも公平に扱う。ランキングは出さず、条件に合う製品をフィルターで提示する
- **出典の透明性**: データの出所を常に明示する

## 8. 画面構成（想定）

### 8.1 トップページ
- アプリの説明（キーメッセージ）
- 「条件で絞り込む」ボタン → フィルター画面へ
- 「AIに相談する」ボタン → チャット画面へ

### 8.2 比較・フィルター画面
- 条件フィルター（サイドバーまたは上部）: 予算、容量、圧力有無、Wi-Fi、かき混ぜ機能など
- フィルター結果を横断比較テーブルで表示
- 各製品の詳細カードにLayer 1＋Layer 2情報を統合表示

### 8.3 AIチャット画面
- 6社のデータを踏まえた自然言語チャット
- 「3人家族で圧力必須、予算3万円以内」のような質問に対し、条件に合う候補を提示
- 回答にはデータの出典を含める

### 8.4 製品詳細ページ
- 1製品ごとのLayer 1スペック一覧
- Layer 2（SNS・コミュニティ情報、メンテナンス実態）
- 出典リンク一覧
